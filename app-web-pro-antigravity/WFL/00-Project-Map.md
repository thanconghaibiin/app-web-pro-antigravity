Quy tắc:



Output phải dùng Markdown, đúng cấu trúc mục bên dưới.



Tuyệt đối không viết lan man; mỗi bullet tối đa 1–2 dòng.



Nếu thiếu thông tin, tự đưa giả định hợp lý và ghi rõ “Giả định”.



Phải có mục “Issue \& Bug Rules” gồm tag module + severity + quy tắc đặt tên ticket.



Phải có mục “Release \& Versioning” gồm quy tắc version + changelog + rollback.



Kết thúc bằng “DONE CRITERIA” (tiêu chí đạt) để team tick.
